# 代码审计

用 OMC 对已有项目进行全面审计。

## 上线审计

```
/autopilot "审计这个项目是否可以部署上线，检查：
1. API 是否完整实现（非 stub/TODO）
2. 前端页面是否完整可用
3. Docker/部署配置是否就绪
4. 测试是否真实可运行
5. 有无 TODO/FIXME/HACK
6. 安全问题（CORS、密钥泄露、Swagger 暴露）
输出一份带严重度的上线清单"
```

## 审计输出示例

OMC 会启动多个并行 Agent 分别审计不同维度，最终合并为一份报告：

```
总体结论：基本可上线，有 3 项必须修复 + 4 项建议修复

🔴 必须修复（上线阻断）
┌─────┬───────────────────────┬────────┬─────────────────────┐
│  #  │         问题          │ 严重度 │      修复建议       │
├─────┼───────────────────────┼────────┼─────────────────────┤
│  1  │ Swagger 生产环境暴露  │ HIGH   │ NODE_ENV 判断       │
│  2  │ CORS * + credentials  │ HIGH   │ CORS_ORIGIN 环境变量│
│  3  │ CI/生产 Node 版本不一致│ MEDIUM │ 统一为 Node 20      │
└─────┴───────────────────────┴────────┴─────────────────────┘

🟡 建议修复（非阻断）
...

🟢 安全正面发现
...
```

## 审计后修复

```
/ralph "修复审计发现的所有阻断问题并确保测试通过"
```

修复后验证：

```
/ultraqa "验证所有测试通过，确保修复没有引入回归"
```

## 专项审计

### 安全审计
```
/security-review
```

### 代码质量
```
/simplify
```

### 架构审查
```
/ask codex "review 整体架构设计，指出技术债务"
```

### 性能分析
```
/autopilot "分析 API 性能瓶颈：数据库查询 N+1、缺失索引、未优化的分页"
```
