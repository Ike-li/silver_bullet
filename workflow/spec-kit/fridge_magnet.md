## 三条不可妥协（多了就稀释）
                                                                                                                                                                                                                                                                             
  1. **不让用户暴露在你的失败中**                                                                                                                                                                                                                                            
     错误必须给用户可执行的下一步（重试 / 回退 / 求助）；
     外部依赖必须有超时 + 降级；不允许让用户看到堆栈或英文错误码。                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                             
  2. **不让密码 / token / PII 进日志**                                                                                                                                                                                                                                       
     违反一次就是事故，不需要讨论。Logger 必须配 redactor，                                                                                                                                                                                                                  
     redactor 必须有测试。                                                                                                                                                                                                                                                   
  
  3. **不在没测试的情况下改核心数据写入路径**                                                                                                                                                                                                                                
     核心 = 钱、用户身份、数据完整性。                      
     这三类代码 MUST 先有失败的测试再有实现。                                                                                                                                                                                                                                
     其他代码（UI、view、查询）可以先发了再补测试。                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                             
  ## 默认做，但可豁免（豁免要写在 PR 描述里）                                                                                                                                                                                                                                
                                                            
  | 默认 | 豁免场景 |                                                                                                                                                                                                                                                        
  |---|---|                                                 
  | 测试金字塔（unit + integration + e2e） | 一次性脚本 / prototype / 单文件 < 100 行的 UI 微调 |
  | 结构化日志 + trace id | 本地开发模式 |                                                                                                                                                                                                                                   
  | 性能预算守护 | 内部工具 / admin 后台 |                                                                                                                                                                                                                                   
  | WCAG AA | 内部工具 / 运维面板 |                                                                                                                                                                                                                                          
  | design system 复用 | 实验路径，但 14 天内必须收敛或删除 |                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  ## 明确不做（这一节最重要）                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  - **不上微服务**，直到 ≥ 5 个团队需要独立部署节奏                                                                                                                                                                                                                          
  - **不抽 Repository / DAO 接口**，直到有第 2 种数据源
  - **不引第二种 RPC 风格**（不要 REST + GraphQL 混合）                                                                                                                                                                                                                      
  - **不写 abstract base class**，直到出现第 3 个具体子类                                                                                                                                                                                                                    
  - **不引新的状态管理库**，除非有具体的"现有方案做不到 X"的证据
  - **不在生产同步发邮件 / 写审计 / 调外部 API**——一律走队列                                                                                                                                                                                                                 
  - **不为"未来可能的"需求写抽象**——三处重复才允许抽象，两处以下复制                                                                                                                                                                                                         
  - **数字预算不从博客抄**——从生产 P75 数据 + 业务可承受度反推                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  ## 阶段化纪律                                                                                                                                                                                                                                                              
                                                                                                                                                                                                                                                                             
  | 阶段 | 触发条件 | 加什么 |                                                                                                                                                                                                                                               
  |---|---|---|
  | **MVP** | < 6 个月 / < 100 真实用户 | 只守上面三条不可妥协 + "明确不做"清单 |                                                                                                                                                                                            
  | **PMF** | DAU > 1000 或 付费用户 > 100 | 加：覆盖率 ≥ 70% / 性能基准 / 三大可观测性支柱 |                                                                                                                                                                                
  | **规模化** | 团队 > 10 人 或 多区域部署 | 加：SemVer / 迁移指南 / 多人 review / SLO 与 error budget |                                                                                                                                                                    
                                                                                                                                                                                                                                                                             
  升阶不是终点，是仪式——升上去就回不来。该升不升 = 技术债；提前升 = 过度治理。                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  ## AI 协作边界                                                                                                                                                                                                                                                             
                                                            
  - AI 可以：写代码 / 写测试 / 写文档 / 提出方案                                                                                                                                                                                                                             
  - AI 不能单独决定：DB 迁移上生产 / 密钥与权限变更 / 对外接口契约 / 删除任何数据
  - AI 生成的测试 MUST 在合并前由人确认它**真的会 fail**（防恒真测试）                                                                                                                                                                                                       
  - AI 写的 commit 必须由人 review 后合并（哪怕作者也是 AI）                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                             
  ## 修订与过期                                                                                                                                                                                                                                                              
                                                            
  - **每季度复审一次**：删掉 90 天内没拦下任何 PR 的条款。原则不是装饰品。                                                                                                                                                                                                   
  - **加新条款必须附"上次违反它造成的具体损失"**（事故、客诉、回滚记录）。没有损失证据的条款不进。
  - 修订走 PR；不走 Sync Impact Report 那套——commit message 写清楚就够。
