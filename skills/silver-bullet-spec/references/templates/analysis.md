## module-inventory.md

```markdown
# 模块清单

| 模块 | 职责 | 依赖 | 文件数 | 行数 | 复杂度 | S.U.P.E.R 评分 |
|:-----|:-----|:-----|-------:|-----:|:-------|:---------------|
| | | | | | | |

> **S.U.P.E.R 评分**：按五项原则用 🟢（符合）、🟡（部分）、🔴（违规）标注。格式示例：`S🟢 U🟡 P🔴 E🟢 R🟡`

## 模块明细

### <模块名>
- **路径**：`src/module_name/`
- **职责**：该模块做什么
- **对外 API**：暴露给其他模块的主要函数/类
- **内部依赖**：引用了本仓库哪些模块
- **外部依赖**：使用的第三方库
- **复杂度**：低 / 中 / 高 / 严重
- **改造说明**：本模块在本次任务中的难点与注意点
- **S.U.P.E.R 评估**：
  - **S（单一职责）**：是否只做一件事？否则如何拆？
  - **U（单向流动）**：依赖是否单向？是否有环依赖？
  - **P（端口优于实现）**：输入输出是否契约化、可序列化？边界是否基于契约？
  - **E（环境无关）**：是否有硬编码路径、内嵌配置或平台假设？
  - **R（可替换）**：能否替换而不连锁修改？替换成本如何？
```
