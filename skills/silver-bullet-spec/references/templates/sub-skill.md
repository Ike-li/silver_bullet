# 任务执行 SKILL 模板（阶段 5）

在仓库内生成**任务级执行 skill**，路径为：

`skills/<task-slug>/SKILL.md`

- **`<task-slug>`**：小写、连字符，在 `skills/` 下唯一（例如 `api-migration-2026`）。
- 本工作流**不要**把 skill 安装到全局或仓库外。

用手写方式编写该文件：标准 YAML frontmatter（`name`、`description`）加下文各节。实施阶段的并行约定见仓库根路径 `skills/silver-bullet-spec/references/parallel-protocol.md`。

## 内容大纲（按此顺序）

### 2. S.U.P.E.R 架构原则（必须内联）

**不要**仅链接到 `super-philosophy.md`。须将下述块**原样嵌入**生成的 `SKILL.md`，使实施者无需打开其他文件即可看到原则：

```markdown
## S.U.P.E.R 架构 — 强制编码标准

> 像搭乐高一样写代码：每块只做一件事、接口标准、流向清晰、随处可跑、随时可换。

本仓库产出的代码**必须**符合以下五项原则；违反视为缺陷。

### S — 单一职责
- 每个模块、文件、函数只解决一个问题
- 倾向拆分；组合产生能力
- **试金石**：能否用一句话说清本模块职责？不能则拆分。

### U — 单向流动
- 数据单向：输入 → 处理 → 输出
- 依赖向内：外层依赖内层，内层不感知外层
- 禁止循环引用与反向依赖
- **试金石**：核心逻辑能否在零外部服务下跑单元测试？

### P — 端口优于实现
- 先定义接口契约（JSON Schema、类型、数据结构），再写实现
- 跨模块 I/O 必须可序列化
- 替换数据源、渲染层、通知渠道时，核心逻辑零改动
- **实践**：模块边界一律使用显式、schema 化的契约

### E — 环境无关
- 配置经环境变量或配置文件，禁止硬编码
- 依赖在清单中声明（requirements.txt / package.json / Cargo.toml 等）
- 进程无状态；持久化交给外部存储
- 日志走标准输出；同一套代码可在本机、Docker、云端运行
- **配置优先级**：环境变量 > .env > 配置文件 > 代码默认值

### R — 可替换部件
- 任一层可替换而不牵连其他层
- **替换成本**是架构质量的核心指标
- 若替换一处导致架构级连锁破坏，则设计失败
- **自检**：每个模块自问「能否只改本模块目录就换成另一种实现？」
```

### 3. S.U.P.E.R 代码评审清单（每项任务后）

包含下列清单，并在勾选任务完成前要求执行：

```markdown
## S.U.P.E.R 代码评审 — 每项任务后执行

勾选「完成」前，须逐项确认：

| # | 检查项 | 原则 | 通过？ |
|:--|:-------|:-----|:------|
| 1 | 每个新模块/文件职责单一 | S | |
| 2 | 无函数承担多个概念职责 | S | |
| 3 | 数据流为 输入→处理→输出，无反向依赖 | U | |
| 4 | 未引入循环引用 | U | |
| 5 | 跨模块接口为契约化（类型/接口） | P | |
| 6 | 模块 I/O 可序列化 | P | |
| 7 | 无硬编码路径、URL、密钥或配置值 | E | |
| 8 | 新依赖已在依赖清单中声明 | E | |
| 9 | 新模块可在不改其他模块前提下替换 | R | |
| 10 | 变更后全部测试通过 | — | |

**计分**：全过 → 可继续。1～2 项不过 → 先修再标完成。3 项及以上不过 → 停止并重构。
```

### 4. 目标技术栈编码规范

本仓库的具体约定：命名、目录、错误处理、测试等。新增或修改行为时优先**测试驱动开发**（红—绿—重构），并与项目惯例一致。

### 5. 项目专属架构上下文

来自 `docs/analysis/`，尤其 `risk-assessment.md`：须修复的违规、目标分层、关键契约。

### 6. 进度更新说明

每项任务后：勾选阶段文件；更新 `docs/progress/MASTER.md` 中的计数与 **当前状态** / **下一步** / **最后更新**；标完成前先跑 S.U.P.E.R 清单。

### 7. 并行执行约定

概括泳道用法；全文见 `skills/silver-bullet-spec/references/parallel-protocol.md`（相对仓库根）。要点：遵循 `docs/plan/task-breakdown.md` 中的泳道与合并风险；并行工作后合并并测试。

### 8. 归档触发

全部任务完成后，按主 skill **silver-bullet-spec** 执行阶段 7：将产物移入 `docs/archives/<项目名>/`，布局见 `skills/silver-bullet-spec/references/templates/archive.md`。
