# 行为规则

以下规则适用于 **silver-bullet-spec** 工作流的每个阶段。除非用户明确同意例外并在进度备注中记录，否则不得违反。

---

1. **不得跳过阶段。** 若某阶段看似多余，仍须产出该阶段的轻量版本，以保持可追溯。

2. **阶段边界须确认。** 离开某阶段前，总结状态并取得用户明确同意后再继续（普通对话即可）。

3. **记录决策。** 将决策、权衡与阻塞记入对应 `docs/progress/` 阶段文件的 **备注**（必要时记入会话日志）。

4. **必须更新进度。** 完成任一任务或结束会话后，须更新阶段明细文件的勾选框 **以及** `docs/progress/MASTER.md`（计数、当前状态、下一步、最后更新）。

5. **新会话 = 先读 `docs/progress/MASTER.md`。** 该文件是跨会话的单一事实来源。

6. **尊重用户时间。** 摘要简洁；多用列表与表格，少写长文。

7. **完成后归档。** 全部任务完成时执行阶段 7，将产物保留在 `docs/archives/` — 勿让工作副本散落或悄悄删除历史。

8. **跨会话记忆单一来源。** 以 `docs/progress/` 下 Markdown 为持久化依据；勿假定存在或与某 IDE 任务列表同步。

9. **用户交互。** 提问与确认使用自然对话；勿假定存在特定的问答界面或专用工具。
