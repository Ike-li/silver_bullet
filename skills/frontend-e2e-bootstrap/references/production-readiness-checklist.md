# 前端 E2E Bootstrap 生产就绪检查清单

这份清单用于判断“这套前端测试体系是否已经能进入生产使用”，而不是停留在方案或 demo 阶段。

## 1. 结构

- [ ] 已存在统一的页面级测试目录
- [ ] 已存在 `playwright.config.*` 或等价配置
- [ ] 没有平行的第二套 E2E 目录在互相竞争

## 2. 命令入口

- [ ] 已有类型检查命令，例如 `npm run type-check`
- [ ] 已有 mocked E2E 命令，例如 `npm run test:e2e`
- [ ] 已有 live smoke 命令，例如 `npm run test:e2e:smoke`
- [ ] 命令名称与项目现有脚本风格一致

## 3. 共享 helpers

- [ ] `auth` 已定义 mocked E2E 与 live smoke 的不同登录策略
- [ ] `monitor` 已统一监听 `console error`、`pageerror`、HTTP `>= 400`
- [ ] `mock-api` 已提供成功/失败响应构造与 payload 捕获能力
- [ ] `page-helpers` 已沉淀至少一个公共页面动作

## 4. mocked E2E

- [ ] 已有第一批关键页面回归
- [ ] 每个关键页面至少验证表单、按钮、弹窗或跳转中的一类核心行为
- [ ] 关键请求已做 payload 或 response 契约断言
- [ ] 未覆盖请求不会静默放过

## 5. live smoke

- [ ] 走真实登录，不使用假 token
- [ ] 已覆盖主导航或主链路
- [ ] 关键 API 的 `4xx/5xx` 能被监控到
- [ ] 已定义允许噪音的白名单及理由

## 6. 契约治理

- [ ] 已明确至少一个可靠契约来源
- [ ] mock 字段名、嵌套结构、空值语义与真实契约一致
- [ ] 字段变更时会同步更新 mock、断言和前端解析逻辑

## 7. 输出与交接

- [ ] 每轮都能输出已覆盖页面与未覆盖页面
- [ ] 每轮都能输出失败分类
- [ ] 每轮都能输出风险与下一步
- [ ] 团队成员不看作者口头说明，也能继续扩测试
