# Root Cause Checklist（完成前自检）

在宣称"根因已定位"或"修复已完成"之前，逐项确认：

## 定位阶段

- [ ] 完整阅读了报错、堆栈、日志关键信息，记录了文件、行号、错误码
- [ ] 有可重复触发的复现步骤（或明确标注"暂不可稳定复现"及缺失证据）
- [ ] 检查了近期代码/配置/依赖/环境变化，列出候选变更
- [ ] 能说明"异常首次出现在哪一段链路/哪个边界"

## 验证阶段

- [ ] 当前假设只有一个，且有明确的证据支持
- [ ] 验证动作是最小化的（最小改动、最小输入、最短反馈）

## 修复阶段

- [ ] 修复仅针对根因，未顺手做无关变更
- [ ] 原问题已不再出现
- [ ] 相关测试/构建通过，未引入新回归
