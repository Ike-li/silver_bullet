# Root Cause Checklist（轻量）

用于快速确认本轮调试是否具备“先根因后修复”的最小条件。

## 1) 错误信息检查

- 是否完整阅读报错、堆栈、日志关键信息？
- 是否记录了关键文件、行号、错误码或异常类型？

## 2) 复现确认

- 是否有可重复触发步骤（输入、环境、频率）？
- 若暂不可稳定复现，是否明确缺失证据而非直接猜测？

## 3) 最近改动检查

- 是否检查了近期代码/配置/依赖/环境变化？
- 是否列出“与问题时间相关”的候选变更？

## 4) 多组件边界采证

- 是否在组件边界确认了输入/输出与状态？
- 是否能说明“异常首次出现在哪一段链路”？

## 5) 单一假设验证

- 是否明确了单一假设（而非多个同时猜）？
- 是否用最小验证动作测试该假设？
- 是否避免一次改多个变量？

## 6) 修复后回归验证

- 若是 bugfix：原问题是否已不再出现？
- 若是实现任务：相关测试/构建/验证是否通过且输出符合预期？
- 验证失败或不明确时，是否避免标记为完成？
